# TypeScript 代码规范（基于主流大厂标准）

本文档基于 **Google**、**Microsoft**、**Airbnb** 等主流大厂的 TypeScript 编码规范整理。

---

## 1. 文件命名规范

### 1.1 基本规则

| 规范来源            | 推荐格式                               | 示例                                 |
| ------------------- | -------------------------------------- | ------------------------------------ |
| **Google**          | `snake_case`（蛇形命名）               | `user_service.ts`、`data_manager.ts` |
| **Airbnb/现代前端** | `kebab-case`（短横线命名）             | `user-profile.component.ts`          |
| **特殊情况**        | `PascalCase`（仅当文件只导出一个类时） | `AuthService.ts`                     |

### 1.2 详细说明

**✅ 推荐做法：**
```typescript
// Google 风格
user_service.ts
national_festival_module.ts
data_manager.ts

// Airbnb/现代前端风格
user-profile.component.ts
national-festival.module.ts
data-manager.service.ts
```

**❌ 不推荐：**
```typescript
// 避免使用 PascalCase（除非文件只导出一个类）
UserService.ts  // 不推荐
NationalFestivalModule.ts  // 不推荐
```

**⚠️ 特殊情况：**
- 当文件仅导出一个类，且文件名与类名完全一致时，可以使用 `PascalCase`
- 例如：`AuthService.ts` 导出 `class AuthService`

### 1.3 文件扩展名

- TypeScript 文件：`.ts`
- TypeScript React 文件：`.tsx`
- 类型定义文件：`.d.ts`

## 2. 类、枚举、接口的声明

### 2.1 类（Class）

**命名规范：** 使用 `PascalCase`（大驼峰命名）

```typescript
// ✅ 正确
class UserManager {
  // ...
}

class NationalFestivalData {
  // ...
}

// ❌ 错误
class userManager { }  // 首字母应大写
class user_manager { }  // 不应使用下划线
```

**声明规范：**
- 类名应使用名词，描述"它是什么"
- 类名应清晰、具体，避免缩写（除非是通用缩写）

### 2.2 接口（Interface）

**命名规范：** 使用 `PascalCase`（大驼峰命名）

```typescript
// ✅ 正确
interface User {
  id: number;
  name: string;
}

interface ModuleData {
  // ...
}

// ❌ 错误 - 禁止使用 I 前缀
interface IUser { }  // Google 和 TypeScript 官方禁止
interface IModuleData { }  // 不推荐
```

**重要规则：**
- **禁止使用 `I` 前缀**（如 `IUser`）
- 原因：接口和类在 TypeScript 中常可互换使用，前缀增加视觉噪音
- 接口名称应描述"它是什么"或"它能做什么"

### 2.3 枚举（Enum）

**枚举名：** 使用 `PascalCase`

**枚举值：** 使用 `CONSTANT_CASE`（全大写下划线）

```typescript
// ✅ 正确 - Google 推荐
enum ColorType {
  RED_LIGHT = 1,
  BLUE_DARK = 2,
  GREEN_BRIGHT = 3,
}

enum HandBookItemState {
  UN_ACTIVE = 0,
  NORMAL = 1,
  SELECT = 2,
}

// ❌ 错误
enum ColorType {
  redLight = 1,  // 不应使用 camelCase
  BlueDark = 2,  // 不应使用 PascalCase
}
```

**说明：**
- 枚举名使用 `PascalCase`
- 枚举值使用 `CONSTANT_CASE`（全大写 + 下划线）
- 这是 Google 和大多数大厂的推荐做法

### 2.4 类型别名（Type Alias）

**命名规范：** 使用 `PascalCase`

```typescript
// ✅ 正确
type UserId = number;
type ConfigMap = Map<string, string>;
type EventHandler = (event: Event) => void;
```

### 2.5 泛型参数（Generic Parameters）

**命名规范：** 单个大写字母或 `PascalCase`

```typescript
// ✅ 正确
class Container<T> { }
class Container<TItem> { }
function map<T, U>(items: T[], fn: (item: T) => U): U[] { }
```

## 3. 类成员变量、成员方法的声明

### 3.1 命名规范

**基本规则：** 统一使用 `camelCase`（小驼峰命名）

```typescript
// ✅ 正确
class UserService {
  userName: string;
  isActive: boolean;
  maxRetryCount: number;
  bgSelected: Sprite;  // 不是 bg_Selected
  nodeUnActive: Node;  // 不是 node_UnActive
}

// ❌ 错误
class UserService {
  user_name: string;  // 不应使用下划线
  bg_Selected: Sprite;  // 不应在中间使用下划线
  node_UnActive: Node;  // 不应在中间使用下划线
}
```

**禁止规则：**
- ❌ 禁止在变量名中间使用下划线（如 `bg_Selected` → `bgSelected`）
- ❌ 禁止使用 `snake_case` 命名变量和方法

### 3.2 可见性修饰符（Visibility Modifiers）

#### 3.2.1 公开成员（Public）

**规则：** **必须省略 `public` 关键字**

```typescript
// ✅ 正确 - Google/Microsoft/Airbnb 规范
class UserService {
  userName: string;  // 省略 public，默认就是 public
  isActive: boolean;
  
  getUserName(): string {
    return this.userName;
  }
}

// ❌ 错误 - 冗余的 public
class UserService {
  public userName: string;  // 不应显式写 public
  public getUserName(): string { }  // 不应显式写 public
}
```

**原因：**
- TypeScript 默认所有成员都是 `public`
- 显式写出 `public` 是冗余的，增加视觉噪音
- Google 规范强制要求省略

#### 3.2.2 私有成员（Private）

**规则：** 使用 `private` 关键字，**禁止使用 `_` 前缀**

```typescript
// ✅ 正确 - Google/Airbnb 规范
class UserService {
  private retryCount: number = 0;
  private dataCache: Map<string, any> = new Map();
  
  private calculateTotal(): number {
    // ...
  }
}

// ❌ 错误 - 不应使用 _ 前缀
class UserService {
  private _retryCount: number = 0;  // 不应使用 _ 前缀
  private _dataCache: Map<string, any>;  // 不应使用 _ 前缀
}
```

**原因：**
- `private` 关键字已经明确表达了可见性
- `_` 前缀是冗余的，不符合主流规范
- Google 和 Airbnb 都禁止使用 `_` 前缀

**例外情况：**
- 仅当存在同名 getter/setter 时，为了避免命名冲突，可以使用 `_` 前缀：
```typescript
class User {
  private _name: string;  // 允许：因为存在 getter
  get name() { return this._name; }
  set name(value: string) { this._name = value; }
}
```

#### 3.2.3 受保护成员（Protected）

**规则：** 使用 `protected` 关键字

```typescript
// ✅ 正确
class BaseService {
  protected config: Config;
  protected initialize(): void { }
}
```

### 3.3 只读属性（Readonly）

**规则：** 如果属性在构造函数初始化后不再改变，必须使用 `readonly`

```typescript
// ✅ 正确
class UserService {
  private readonly maxRetries: number = 3;
  private readonly logger: Logger;
  
  constructor(logger: Logger) {
    this.logger = logger;
  }
}
```

### 3.4 构造函数参数属性（Parameter Properties）

**规则：** 推荐使用参数属性简化代码

```typescript
// ✅ 推荐 - 简洁写法
class UserService {
  constructor(
    private readonly db: Database,
    private readonly logger: Logger
  ) {}
}

// ❌ 不推荐 - 冗余写法
class UserService {
  private readonly db: Database;
  private readonly logger: Logger;
  
  constructor(db: Database, logger: Logger) {
    this.db = db;
    this.logger = logger;
  }
}
```

**注意：** 在构造函数参数属性中，`public` 关键字是允许的（这是唯一允许显式使用 `public` 的场景）：
```typescript
class User {
  constructor(public name: string) {}  // 允许：参数属性
}
```

### 3.5 方法命名

**规则：** 使用 `camelCase`，方法名应为动词

```typescript
// ✅ 正确
class UserService {
  getUserById(id: number): User { }
  updateUser(user: User): void { }
  deleteUser(id: number): boolean { }
  isUserActive(id: number): boolean { }
}

// ❌ 错误
class UserService {
  GetUserById(id: number): User { }  // 不应使用 PascalCase
  get_user_by_id(id: number): User { }  // 不应使用下划线
}
```

### 3.6 Getter 和 Setter

**规则：** 如果只是简单的访问器，建议直接使用公共属性

```typescript
// ✅ 推荐 - 简单情况直接使用属性
class User {
  name: string;  // 直接公开
  readonly id: number;  // 只读属性
}

// ✅ 允许 - 需要逻辑处理时使用 getter/setter
class User {
  private _name: string;
  get name(): string {
    return this._name.toUpperCase();
  }
  set name(value: string) {
    this._name = value.trim();
  }
}
```

## 4. 常量命名

**规则：** 使用 `CONSTANT_CASE`（全大写下划线）

```typescript
// ✅ 正确
const MAX_RETRY_COUNT = 3;
const DEFAULT_TIMEOUT = 5000;
const API_BASE_URL = 'https://api.example.com';

// ❌ 错误
const maxRetryCount = 3;  // 不应使用 camelCase
const MaxRetryCount = 3;  // 不应使用 PascalCase
```

## 5. 总结对比表

| 类型          | 命名规范                               | 示例                    | 禁止做法                      |
| ------------- | -------------------------------------- | ----------------------- | ----------------------------- |
| **文件名**    | `snake_case` 或 `kebab-case`           | `user_service.ts`       | `UserService.ts`（除非是类）  |
| **类名**      | `PascalCase`                           | `UserManager`           | `userManager`、`user_manager` |
| **接口名**    | `PascalCase`（无 `I` 前缀）            | `User`                  | `IUser`                       |
| **枚举名**    | `PascalCase`                           | `ColorType`             | `colorType`                   |
| **枚举值**    | `CONSTANT_CASE`                        | `RED_LIGHT`             | `redLight`、`RedLight`        |
| **变量/方法** | `camelCase`                            | `userName`、`getUser()` | `user_name`、`bg_Selected`    |
| **私有成员**  | `private` + `camelCase`（无 `_` 前缀） | `private retryCount`    | `private _retryCount`         |
| **公开成员**  | 省略 `public` + `camelCase`            | `userName`              | `public userName`             |
| **常量**      | `CONSTANT_CASE`                        | `MAX_COUNT`             | `maxCount`                    |

## 6. ESLint 配置建议

如果想通过工具强制规范，可以配置以下 ESLint 规则：

```json
{
  "rules": {
    // 禁止显式使用 public
    "@typescript-eslint/explicit-member-accessibility": [
      "error",
      { "accessibility": "no-public" }
    ],
    // 禁止使用下划线前缀/后缀
    "no-underscore-dangle": ["error", {
      "allow": [],
      "allowAfterThis": false,
      "allowAfterSuper": false
    }],
    // 命名规范
    "@typescript-eslint/naming-convention": [
      "error",
      {
        "selector": "class",
        "format": ["PascalCase"]
      },
      {
        "selector": "interface",
        "format": ["PascalCase"],
        "custom": {
          "regex": "^I[A-Z]",
          "match": false
        }
      },
      {
        "selector": "variable",
        "format": ["camelCase", "UPPER_CASE", "PascalCase"]
      },
      {
        "selector": "memberLike",
        "modifiers": ["private"],
        "format": ["camelCase"],
        "leadingUnderscore": "forbid"
      }
    ]
  }
}
```
