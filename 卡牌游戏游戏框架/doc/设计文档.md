# 回合制卡牌战斗框架 · 设计文档

> 引擎：Cocos Creator 3.8.5 + TypeScript　|　架构：方案 A（数据驱动 + 事件驱动的 Effect / Buff 管线）

## 1. 设计目标（对应需求）

| 需求 | 落地方式 |
|------|----------|
| ① 回合制卡牌战斗 | `flow/battle.ts` 回合主循环 |
| ② 双方 1–5 站位 + 我方独立主动技能 | `entity/team.ts` 站位 + `command/player-skill.ts` |
| ③ 按速度排序逐个出手、一方全灭判负 | `Battle.buildActionOrder` + `Battle.checkEnd` |
| ④ 战斗逻辑与 UI 分离、可服务器校验 | `battle-core`（零 `cc` 依赖）+ 确定性随机 + 战斗记录回放 |
| ⑤ 通用、易扩展武将/技能/Buff | 原子 `Effect` + 事件触发 `Buff`，扩展只加配置 + 小类 |

## 2. 分层架构

```
配置(sample-data) → battle-core(纯逻辑) → BattleRecord(帧) → battle-view(cc回放)
                                  ↘ 服务器用同一份 battle-core 跑一遍做校验 ↙
```

- **逻辑层 `scripts/battle-core/`**：纯 TypeScript，**禁止 import 'cc'**，可在 Node / 服务器直接运行。
- **表现层 `scripts/battle-view/`**：依赖 `cc`，只读取战斗记录逐帧播放动画，**不做任何战斗计算**。
- **数据 `scripts/data/`**：示例武将/技能/Buff 配置，生产可替换为 JSON 表或服务器下发。

## 3. 逻辑层模块

| 目录 | 职责 |
|------|------|
| `core/seeded-random.ts` | 确定性随机（mulberry32），战斗可重放、可校验的基础 |
| `core/event-bus.ts` | 战斗事件总线，Buff/被动靠订阅事件触发 |
| `core/battle-context.ts` | 战斗世界状态中心 + 数值结算入口（伤害/治疗/加 Buff/死亡） |
| `config/config-types.ts` | 全部数据结构与枚举定义 |
| `entity/hero.ts`、`team.ts` | 武将（有效属性 = 基础值经 Buff 修正）、队伍 |
| `skill/skill.ts`、`target-selector.ts` | 技能实例、目标选择策略 |
| `effect/*` | 原子效果接口 + 注册表 + 伤害/治疗/加 Buff |
| `buff/*` | Buff 实例与容器（叠加规则、属性聚合、回合衰减） |
| `flow/battle.ts` | 战斗流程编排（begin/step/run/finish），委托子模块执行 |
| `flow/player-command-manager.ts` | 玩家主动技能指令队列与结算 |
| `flow/round-resolver.ts` | 回合内武将出手（排序后逐个行动、AI 决策、Buff 衰减） |
| `flow/action-order.ts` | 速度排序（纯函数） |
| `flow/battle-outcome.ts` | 胜负判定（纯函数） |
| `flow/battle-factory.ts` | 从 `BattleSetup + GameConfig` 装配并运行战斗 |
| `command/player-skill.ts` | 我方独立于武将之外的主动技能 |
| `record/battle-record.ts` | 战斗记录帧定义与快照 |

## 4. 核心机制

### 4.1 确定性（服务器校验前提）
战斗是 `初始数据 + seed + 玩家指令序列` 的纯函数。所有随机走 `SeededRandom`。相同输入 ⇒ 逐帧完全相同的 `BattleRecord`。服务器用同一份 `battle-core` 重跑（`runBattleWithCommands`），对比客户端上报的 `seed/指令/结果` 即可反作弊。

### 4.2 技能 = 原子效果组合
技能配置里 `effects` 是一组原子效果（`DAMAGE / HEAL / ADD_BUFF`）。新增玩法 = 新增一个 `Effect` 实现并在 `battle-factory.createEffectRegistry` 里 `register`，**不改动主流程**。

### 4.3 Buff = 事件触发器 + 管线修正器
Buff 可携带：
- `attributeModifiers`（属性增减）、`preventAction`（控制类）；
- `triggerTiming + triggerEffects`（如中毒在回合开始掉血），`BattleContext.processBuffTriggers` 在对应时机执行；
- **战斗管线修正**：`damageDealtPercent / damageTakenPercent / critChanceBonus / lifestealPercent / reflectPercent / shield / healingTakenPercent / preventDeath`。增益/减益/护盾/吸血/反伤/免死等都靠配这些字段实现，无需写代码。

### 4.3.1 伤害结算管线（`combat/damage-pipeline.ts`）
所有伤害统一走 `BattleContext.dealDamage(request)` → 管线，按固定顺序应用各阶段修正：
```
暴击 → 攻方增伤 → 减防 → 受方易伤/减伤 → 护盾吸收 → 扣血 → 吸血/反伤 → 死亡/免死
```
- `DamageEffect` 只负责算"基础伤害"（攻击×倍率+固定值）后投入管线；
- 各阶段修正从 `caster/target` 的 Buff 聚合而来（见 `BuffManager.sumXxx`）；
- 治疗经 `applyHeal` 应用 `healingTakenPercent`；
- **扩展点**：加护盾/减伤/吸血/反伤=在 Buff 配字段；要加全新阶段才需动管线本身。

### 4.4 回合流程
每回合：回合开始 → 我方主动技能能量+回合恢复、**结算玩家排队的主动技能指令** → 全体存活武将按 `速度↓、我方优先、站位↑` 排序 → 逐个出手（能量满放大招，否则普攻攒能量）→ 每次行动后判胜负 → 回合结束衰减 Buff。

### 4.5 玩家主动技能（手动释放，需求②）
我方携带的主动技能（`playerActiveSkillIds`）由独立能量驱动、能量随回合增长，攒满后**玩家点击释放**：
- 点击 = `Battle.enqueuePlayerSkill(skillId, targetId?)`，仅排队；
- 真正释放在下一次 `stepBeginRound` 的 `commitPendingCommands`（扣能 + 跑效果 + 写 `commandLog`）；
- 释放时机记为"第几回合"这一确定逻辑坐标，保证可回放/校验；
- 两种驱动共用同一内核：客户端用 `begin/stepBeginRound/stepResolveRound/finish` 交互推进；服务器/回放用 `run(commandLog)` 一次性跑完。

## 5. 战斗记录（帧）
逻辑层输出 `BattleFrame[]`，类型见 `FrameType`（回合开始、出手、伤害、治疗、加/移除 Buff、能量变化、死亡、结束等）。首帧 `BATTLE_START` 带 `snapshot`，供表现层创建武将节点。

## 6. 如何扩展

- **加武将**：在 `data/sample-data.ts` 的 `heroes` 加一条，引用已有/新技能 id。
- **加技能**：在 `skills` 加一条，用现有原子效果组合即可；需要新机制时加一个 `Effect` 并注册。
- **加 Buff**：在 `buffs` 加一条，配置属性修正或触发效果。
- **加效果类型**：实现 `Effect` 接口 → 在 `createEffectRegistry` 注册 → 在 `EffectType` 增加枚举。
- **加战斗机制（护盾/减伤/吸血/反伤/免死/易伤）**：在对应 Buff 配 `damageDealtPercent / damageTakenPercent / shield / lifestealPercent / reflectPercent / preventDeath / healingTakenPercent` 即可，无需改管线。
- **加技能表现（特效/动画/音效/震屏）**：在 `data/skill-view-config.ts` 加一条 `skillId → 表现配置`，`BattlePlayer` 自动查表，无需改播放器代码。

## 7. 运行方式

1. Cocos 编辑器打开项目，打开 `卡牌游戏/game.scene`。
2. 选中 `Canvas` 节点，添加组件 `BattleEntry`（`battle-view/battle-entry.ts`）。
3. 运行预览：左侧蓝色为我方、右侧红色为敌方，逐回合推进回放。
4. 底部是我方主动技能按钮（能量条 + 按钮）。能量随回合增长，攒够后按钮亮起可点：
   - `集火指令`（单体）：点按钮 → 进入选目标 → 点一个敌人确定，下回合开始生效；
   - `战意号令`（群体治疗）：点按钮即排入，下回合开始生效。

> 逻辑层可脱离 Cocos 单独测试：用 esbuild 打包 `battle-core` + `sample-data`，调用 `runBattle(config, setup)` 或 `runBattleWithCommands(config, setup, commandLog)` 即可在 Node 跑通并校验确定性与回放一致。
